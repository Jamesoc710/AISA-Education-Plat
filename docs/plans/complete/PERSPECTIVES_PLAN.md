# Trend Perspectives, Execution Plan

> The content layer that adds a "Perspectives" section to each trend brief: a few genuinely
> conflicting viewpoints, generated by a verified multi-agent pipeline, rendered as a click-to-expand
> accordion on the detail page. Validated on a 2-trend prototype (AI Safety Evaluation Crisis and Big
> Tech Capex vs Revenue), both of which came back "multiple" (not binary), confirming the shape model.
> Branch: `feat/trend-perspectives` (stacked on `feat/trend-redesign`, since the accordion lives on the
> redesigned detail page). Zero em dashes and en dashes in any member-facing copy, as everywhere.

## 1. The shape: four stages, two of them in another chat

```
GENERATE  ->  CURATE  ->  STORE  ->  RENDER
(other chat,   (you, in    (seed,     (detail-page
 ultracode)    the draft)   this repo)  accordion)
```

Generation is the expensive, isolated part (run it in a separate chat). The other three are normal
repo work. The handoff between the two chats is one committed file (`prisma/seed-data/trend-perspectives.ts`),
so the chats never need to talk directly. The grid and detail build fine before the data exists; the
Perspectives section simply does not render until a trend has been seeded (same graceful pattern as themes).

## 2. The data contract

One JSON object per trend, stored in a new additive `perspectives Json?` column on `Trend`
(nullable; `null` means the gate said "skip", so no section renders):

```ts
type Perspectives = {
  shape: "binary" | "multiple" | "tradeoff";
  intro: string;                 // one-sentence framing of the fight (may be empty)
  stances: {
    label: string;               // trend-specific, e.g. "The bubble case"
    who: string;                 // attribution: camp / archetype / verified orgs
    summary: string;             // ONE sentence, shown in the collapsed box
    body: string;                // the fuller reasoning, shown when expanded
    sources: { title: string; url: string }[]; // 0-2 real links, "further reading"
  }[];
  leans: string;                 // honest synthesis, or "genuinely contested"
};
```

## 3. GENERATE, the other-chat runbook

The pipeline is already built and validated: `scripts/workflows/perspectives.workflow.js`. It runs,
per trend, survey -> adversarial shape gate -> per-camp research (dynamic fan-out) -> camp verify ->
synthesize (collapsed one-liner + full body) -> final-copy verify. Models: sonnet researches, opus
judges/verifies, opus writes (swap synth to `fable` once it is available again if you prefer its voice).

**Use a regular Claude Code chat with the `ultracode` keyword. Not deep research** (that is a single
one-question report harness; this purpose-built Workflow is strictly better for a 22-way structured
batch). Max effort is optional: the quality lives in the workflow's per-stage models, not the
orchestrating chat's effort.

Paste this into the other chat (be on the `feat/trend-perspectives` branch):

```
ultracode

Generate the Perspectives content for all trends, end to end:

1. Run:  npx tsx scripts/extract-trends-for-perspectives.ts
   (writes docs/research/perspectives-input.json, an array of 22 trends)

2. Read docs/research/perspectives-input.json, then invoke the Workflow tool with:
      scriptPath: "scripts/workflows/perspectives.workflow.js"
      args:       <the parsed array from that file>
   It runs in the background (~6 stages x 22 trends, ~20-30 min). Wait for it.

3. When it completes, write the results to prisma/seed-data/trend-perspectives.ts as:
      export const TREND_PERSPECTIVES: Record<string, Perspectives | null> = { "<slug>": <perspectives or null>, ... }
   Store null for any trend whose shape is "skip". Keep each trend's rationale / campVerify /
   finalVerify in a sibling docs/research/trend-perspectives.raw.json for the curation pass.

4. Flag for me any trend where finalVerify.ok is false or dashClean is false, with the issues.
   Then commit perspectives-input.json, trend-perspectives.ts, and the raw json.
```

Cost note: roughly 5M tokens, 130 to 165 agents, 20 to 30 minutes. That is a real ultracode spend, which
is why it is gated behind a deliberate run.

## 4. CURATE

Review `prisma/seed-data/trend-perspectives.ts` in place. The pipeline's `finalVerify` already checks
the shipped copy (claims supported, attributions accurate, sources real, dash-clean), so curation is a
light editorial pass: tighten labels, confirm any flagged figures, adjust which sources show. The
cron's `curatedAt` logic protects what you edit once it is live.

## 5. STORE, the build steps (this repo)

- `prisma/schema.prisma`: add `perspectives Json?` to `Trend`; `prisma db push` (additive, like themes).
- `lib/trends.ts`: parse and surface `perspectives` through `getTrendDetail` only (defensive parse, like
  `relatedConcepts`). The grid does not show it, so `TrendCardData` is untouched.
- `scripts/seed-trend-perspectives.ts`: idempotent and surgical (writes ONLY the `perspectives` column by
  slug), validating the shape and running the dash check before each write. Same pattern as
  `scripts/seed-trend-themes.ts`. Never touches status, the publish gate, the cron, or any other field.

## 6. RENDER, the accordion (this repo)

A new full-width "Perspectives" section on the detail page, between the 2-column split and "Recent
signals" (where the redesign plan section 7 reserved it).

- Build it from native `<details>` / `<summary>` elements, not a JS toggle, so it works with zero JS and
  is keyboard-accessible by default: the accordion IS the no-JS / screen-reader backbone.
- Collapsed (`<summary>`): the stance `label` (bold) + the one-line `summary` (text-2) + a caret that
  rotates on open.
- Expanded: the full `body`, the `who` attribution (small, text-3), and the optional `sources` as
  "further reading" links.
- Around the stack: the `intro` line above; `leans` below, under a quiet "Where the evidence leans" label.
- The boxes are a deliberate exception to the no-cards rule (they are interactive, like the concept chips
  and the admin box earned theirs): restrained styling only, hairline border, `radius-2`, an accent edge
  when open. A reduced-motion-aware height/caret animation is enhancement on top of the native behavior.

## 7. Decisions (locked)

- Pipeline order includes a final-copy verify stage (verifies what ships, not just the raw research).
- Each stance carries a one-sentence `summary` (collapsed view) plus the full `body` (expanded view).
- Accordion uses native `<details>` (no-JS safe).
- Source links per stance: yes, as collapsible "further reading"; best-effort, verified, then curated.
- Synthesis model: opus now (Fable is unavailable); swap to fable later if preferred (one constant in the
  workflow script: `SYNTH_MODEL`).
- All 22 trends run uniformly with the final script (the 2 prototype drafts validated quality only).

## 8. Scope

This plan covers Perspectives only. The same harness and the GENERATE -> CURATE -> STORE -> RENDER pattern
extend cleanly to the other planned sections (the trimmed "What's happening now", the two implications
lenses, and the bespoke Spotlight) when you are ready to build those.
